Que s'est-il passé ?

Votre application a été générée avec angular/cli et notre plugin yeoman @civadis/generator-angular-layout permettant d'intégrer le layout et les librairies Civadis. Vous avez dès lors à disposition un projet angular opérationnel pour communiquer avec votre gateway JHipster.

Que faire maintenant ?

Au boulot ! Pour éviter l'angoisse de la page blanche, ce projet ajoute un exemple de page liste/détail, avec un fake service REST. Cet exemple est accessible via le point de menu "Page d'exemple". Libre à vous de vous en inspirer pour créer vos propres écrans. Référez-vous à notre référenciel d'ergonomie si vous souhaitez consulter un maximum d'exemple. Inutile de vous préciser que cette page devra être écrasée par votre propre homepage applicative.

Exemple

Chaque nouvelle rubrique de l'application nécessite une nouveau module dans le folder /src/app/pages/ma_rubrique-page.
La rubrique "exemple-page" illustre exactement la façon d'organiser le code, voir : src/pages/exemple-page.

Admettons qu'on souhaite générer une rubrique pour gérer un concept de facture, nous allons (au minimum) devoir générer les fichiers suivants :
  • /src/app/shared/model/facture.ts : une interface typescript définissant le concept de facture
  • /src/app/pages/facture-page/facture-page.module.ts : le module regroupant tous les élements et dépendances nécessaires à cette rubrique
  • /src/app/pages/facture-page/facture-list/facture-list.component.(ts + scss + html) : le composant permettant d'afficher la liste des factures
  • /src/app/pages/facture-page/facture-form/facture-form.component.(ts + scss + html) : le composant permettant de modifier/ajouter le détail d'une facture (le formulaire d'encodage)
  • /src/app/pages/facture-page/facture-page.routes.ts : la définition des url de navigation vers les composants
  • /src/app/pages/facture-page/facture.service.ts : le service permettant d'accéder aux ressources REST du gateway
NB : Ces fichiers peuvent être générés avec les commandes angular/cli :
  • Générer le module : aller dans src/main/app/pages/ et jouer ng g m facture-page
  • Générer le composant form : aller dans src/main/app/pages/facture-page/ et jouer ng g c facture-form
  • Générer le composant list : aller dans src/main/app/pages/facture-page/ et jouer ng g c facture-list
  • Générer le service : aller dans src/main/app/pages/ et jouer ng g s facture
  • Générer l'interface : aller dans src/main/app/shared/model/ et jouer ng g i facture
Il faudra aussi modifier certains fichiers existants :
  • Référencer facture-page.routes.ts dans le routage angular app-routing.module.ts
  • Ajouter le nouveau point de menu facture dans menu-left.component.ts
  • Ajouter le nouveau module FacturePageModule dans PagesModule
